# 2Brain — IELTS Academic Second Brain

## Project Identity

This vault (`D:\2Brain`) is an **Obsidian second brain** dedicated to preparing for the **IELTS Academic** exam.

- **Current level:** approximately Band 3.0
- **Target level:** Band 5.0
- **Exam date:** July 20
- **Daily budget:** 3 hours of focused study

The vault combines structured notes (vocabulary, grammar, skills, mock tests) with Claude Code skills and rules so that every study session produces durable, reviewable knowledge.

## Vault Philosophy

1. **Capture → Organize → Review.** Every new word, error, or insight should be captured in the right folder, linked to related notes, and revisited regularly.
2. **Band-appropriate explanations.** Explanations should be simple, clear, and targeted at Band 5.0 — avoid native-level complexity unless explicitly requested.
3. **Progress over perfection.** Small daily improvements are more valuable than perfect notes.
4. **Everything in `english/` (except this file).** All English-learning content lives under `english/`. `00_Inbox` is the only exception for quick, unprocessed captures.

## Folder Structure

```
english/
├── 00_Inbox/           # Quick captures (process within 24h)
├── 01_Study_Plan/      # Schedule, daily schedule, progress tracker
├── 02_Vocabulary/      # Topic-based and band-based word banks
├── 03_Grammar/         # Grammar rules with practice
├── 04_Skills/          # Listening, Reading, Writing, Speaking strategies
├── 05_Error_Log/       # Personal mistakes and corrections
├── 06_Mock_Tests/      # Practice test results and analysis
└── 07_Templates/       # Obsidian note templates
```

## How Claude Code Should Help

When working in this vault, Claude Code should:

- Place all new English notes in the correct `english/` subfolder.
- Add the required YAML frontmatter to every note in `english/`.
- Prefer simple explanations and Band 5.0 examples.
- Suggest a daily review when the user has created or modified English notes.
- Help generate vocabulary lists, essay feedback, speaking cue cards, and progress updates using the custom skills.

## Tone

Encouraging but rigorous. Be direct about mistakes, but frame them as learning opportunities. Keep explanations short enough to review quickly.

## Forbidden Actions

- Never delete study notes unless explicitly requested.
- Never overwrite `Progress_Tracker.md` without backing up the previous version in the note history.
- Never create loose English notes in the vault root; use `english/00_Inbox` for quick captures.
- Never use overly complex language when simple language will communicate the idea at Band 5.0.
